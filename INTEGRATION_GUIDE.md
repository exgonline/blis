# BLIS Integration Guide

**Building Load Intelligence Service — v2.0.0**

This document describes how third-party applications connect to BLIS, including authentication, all endpoints, request formats, and every possible response shape.

---

## Base URL

```
https://<your-render-service>.onrender.com/v1
```

All endpoints are prefixed with `/v1`. The version prefix is configurable via the `API_VERSION` environment variable on the server.

---

## Authentication

All endpoints except `GET /v1/health` require an API key.

**Header:**

```
x-blis-api-key: <your-api-key>
```

- The key is a hex string generated by the server (`crypto.randomBytes(32).toString('hex')`).
- It is SHA-256 hashed before being looked up — the raw value is never logged or stored.
- The server records `last_used_at` on every authenticated request.

**Missing or invalid key → 401:**

```json
{
  "error": "UNAUTHORIZED",
  "message": "Missing or invalid API key",
  "statusCode": 401,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

**Inactive key → 401:**

```json
{
  "error": "UNAUTHORIZED",
  "message": "Invalid or inactive API key",
  "statusCode": 401,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

---

## Error Response Format

All error responses share the same envelope:

| Field       | Type   | Description                                           |
|-------------|--------|-------------------------------------------------------|
| `error`     | string | Machine-readable error code (see table below)         |
| `message`   | string | Human-readable description                            |
| `statusCode`| number | HTTP status code (mirrors the HTTP status)            |
| `timestamp` | string | ISO 8601 UTC timestamp when the error occurred        |

**Error codes and HTTP status mapping:**

| `error` code        | HTTP | When it occurs                                                |
|---------------------|------|---------------------------------------------------------------|
| `VALIDATION_ERROR`  | 400  | Request body or query parameter failed validation             |
| `UNAUTHORIZED`      | 401  | Missing, invalid, or inactive API key                         |
| `NOT_FOUND`         | 404  | Resource does not exist                                       |
| `SITE_NOT_FOUND`    | 404  | Site ID not registered (estimate endpoint specific)           |
| `CONFLICT`          | 409  | Attempt to register a site ID that already exists             |
| `UNPROCESSABLE`     | 422  | Site exists but cannot be processed (e.g. no floor area yet)  |
| `INTERNAL_ERROR`    | 500  | Unexpected server error                                       |

In non-production environments, a `stack` field is also included on 500 errors.

## Typical Integration Flow

```
1. Register site        POST /v1/profile
2. (optional) Check     GET  /v1/profile/:siteId
3. Get current load     GET  /v1/estimate/:siteId
4. Get 24h forecast     GET  /v1/estimate/:siteId/forecast
5. Trigger EPC refresh  POST /v1/profile/:siteId/refresh-epc  (if needed)
```

After registration, BLIS automatically fetches an EPC certificate and computes annual kWh benchmarks in the background. The estimate endpoint is available immediately, using statistical benchmarks while the EPC fetch completes.

Public — no API key required.

Returns service status and database connectivity.

**Request:** No parameters.

**Response 200 — healthy:**


```json

  "timestamp": "2026-03-24T12:00:00.000Z",
  "version": "2.0.0",
  "db": {
    "connected": false
```

### POST /v1/profile

Register a new site. Triggers a background EPC fetch automatically.

**Request body (JSON):**

| Field                 | Type   | Required | Constraints                                          |
|-----------------------|--------|----------|------------------------------------------------------|
| `siteId`              | string | Yes      | 1–100 chars, alphanumeric / underscores / hyphens only |
| `address`             | string | Yes      | 1–500 chars                                          |
| `postcode`            | string | Yes      | Valid UK postcode (e.g. `SW1A 2AA`)                  |
| `siteName`            | string | No       | Up to 255 chars                                      |
| `uprn`                | string | No       | Up to 20 chars — used to look up EPC certificate     |
| `buildingTypeOverride`| string | No       | One of the building type enum values (see below)     |
| `floorAreaOverride`   | number | No       | 1–999,999 m²                                         |
| `buildingAgeOverride` | string | No       | One of the building age enum values (see below)      |

**Building type values:**

| Value                   | Description               |
|-------------------------|---------------------------|
| `hotel`                 | Full-service hotel        |
| `hotel_budget`          | Budget hotel              |
| `housing_association`   | Social/residential        |
| `fleet_depot`           | Depot / workshop          |
| `warehouse_simple`      | Basic warehouse / storage |
| `car_park`              | Car park (no facilities)  |
| `car_park_with_facilities` | Car park with facilities |
| `office_general`        | General office            |
| `retail`                | Retail / shop             |
| `pub_restaurant`        | Pub, restaurant, café     |
| `unknown`               | Not yet classified        |

**Building age values:**

| Value        | Description  |
|--------------|--------------|
| `pre_1970`   | Before 1970  |
| `1970_1990`  | 1970–1990    |
| `1990_2005`  | 1990–2005    |
| `post_2005`  | After 2005   |
| `unknown`    | Not known    |

**Example request:**


### POST /v1/profile/:siteId/refresh-epc

Queue an EPC re-fetch for an existing site. Returns immediately; the fetch runs asynchronously.

**Request body:** Empty.

**Response 202 — queued:**

```json
{
  "message": "EPC refresh queued for site site-hotel-001",
  "siteId": "site-hotel-001",
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

**Response 404 — site not found:**

```json
{
  "error": "NOT_FOUND",
  "message": "Site site-hotel-001 not found",
  "statusCode": 404,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

After the background fetch completes, `GET /v1/profile/:siteId/epc` will reflect the new data, and the annual kWh benchmarks in the profile will be recalculated.

### GET /v1/estimate/:siteId

Get the current building electrical load estimate. Returns a stored (pre-calculated) estimate if one covers the current half-hour period, otherwise calculates fresh and stores the result.

**Path parameter:** `siteId`

**Query parameter:**

| Parameter | Type   | Required | Description                                                        |
|-----------|--------|----------|--------------------------------------------------------------------|
| `at`      | string | No       | ISO 8601 datetime. Must be within 7 days of now. Defaults to now.  |

**Response 200:**

```json
{
  "siteId": "site-hotel-001",
  "estimatedAt": "2026-03-24T12:00:00.000Z",
  "buildingType": "hotel",
  "floorAreaM2": 3500,
  "annualKwhP75": 693750.00,
  "currentEstimate": {
    "estimatedKw": 28.4,
    "estimatedAmpsTotal": 43.2,
    "estimatedAmpsL1": 15.5,
    "estimatedAmpsL2": 13.8,
    "estimatedAmpsL3": 13.9,
    "hhIndex": 24,
    "elexonSeason": "spring",
    "dayType": "weekday",
    "elexonCoefficient": 0.00002184,
    "confidenceLevel": "MANUAL_OVERRIDE",
    "safetyMarginApplied": 0.15
  },
  "dataSource": {
    "epcFetchedAt": "2026-03-24T12:05:00.000Z",
    "annualKwhSource": "MANUAL_OVERRIDE",
    "epcRating": "C"
  }
}
```

**Response field reference:**

| Field                             | Type    | Description                                                                   |
|-----------------------------------|---------|-------------------------------------------------------------------------------|
| `siteId`                          | string  | Site identifier                                                               |
| `estimatedAt`                     | string  | ISO 8601 — when the estimate was calculated                                   |
| `buildingType`                    | string  | Effective building type (override takes precedence)                           |
| `floorAreaM2`                     | number  | Floor area used in the calculation                                            |
| `annualKwhP75`                    | number  | Annual kWh at 75th percentile (the basis for the estimate)                    |
| `currentEstimate.estimatedKw`     | number  | Estimated active power for this half-hour period (kW)                         |
| `currentEstimate.estimatedAmpsTotal` | number | Total three-phase current (A) at 400V, 0.95 PF                             |
| `currentEstimate.estimatedAmpsL1` | number  | Phase L1 current (A)                                                         |
| `currentEstimate.estimatedAmpsL2` | number  | Phase L2 current (A)                                                         |
| `currentEstimate.estimatedAmpsL3` | number  | Phase L3 current (A)                                                         |
| `currentEstimate.hhIndex`         | number  | Half-hour period index 0–47 (0 = 00:00–00:30, 47 = 23:30–00:00)             |
| `currentEstimate.elexonSeason`    | string  | Elexon season (see below)                                                     |
| `currentEstimate.dayType`         | string  | `weekday`, `saturday`, or `sunday` (bank holidays treated as Sunday)          |
| `currentEstimate.elexonCoefficient` | number | Elexon load profile coefficient used for this period                        |
| `currentEstimate.confidenceLevel` | string  | `STATISTICAL`, `EPC_DERIVED`, or `MANUAL_OVERRIDE`                           |
| `currentEstimate.safetyMarginApplied` | number | Fraction added above central estimate (currently `0.15` for all levels)  |
| `dataSource.epcFetchedAt`         | string\|null | ISO 8601 — when EPC data was last fetched, or `null` if not yet fetched |
| `dataSource.annualKwhSource`      | string  | Confidence level used for the annual kWh figure                               |
| `dataSource.epcRating`            | string\|null | EPC energy rating (A–G), or `null` if not available                     |

**Elexon season values:**

| Value         | Period                                        |
|---------------|-----------------------------------------------|
| `winter`      | November – February + pre-spring March        |
| `spring`      | Last Sunday of March → last Sunday of May     |
| `high_summer` | Last Sunday of May → last Sunday of July      |
| `summer`      | Last Sunday of July → end of September        |
| `autumn`      | October                                       |

**Response 400 — invalid `at` parameter:**

```json
{
  "error": "VALIDATION_ERROR",
  "message": "Invalid \"at\" parameter — must be ISO8601",
  "statusCode": 400,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

**Response 400 — `at` out of range:**

```json
{
  "error": "VALIDATION_ERROR",
  "message": "The \"at\" parameter must be within 7 days of now",
  "statusCode": 400,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

**Response 404 — site not registered:**

```json
{
  "error": "SITE_NOT_FOUND",
  "message": "Site site-hotel-001 not found",
  "statusCode": 404,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

**Response 422 — site registered but floor area not available yet:**

```json
{
  "error": "UNPROCESSABLE",
  "message": "Floor area not available for site site-hotel-001 — cannot calculate estimate",
  "statusCode": 422,
  "timestamp": "2026-03-24T12:00:00.000Z"
}
```

## Calculation Notes

These apply when interpreting estimate values:

**Power to amps:**
```
I = (kW × 1000) / (√3 × 400V × 0.95)
```
Total amps = three-phase total at 400V line-to-line, power factor 0.95.

**Phase distribution:**
`l1Amps + l2Amps + l3Amps = estimatedAmpsTotal` (approximately — values are rounded individually).
Phase factors are set per building type and sum to 1.0.

**Annual kWh P75:**
```
annualKwhP75 = floorArea × cibse_typical_kwh_per_m2 × age_multiplier × 1.15
```
The 1.15 factor represents the 75th percentile uplift applied to the CIBSE typical benchmark.

**Half-hour period to kW:**
```
kW = annualKwhP75 × elexonCoefficient / 0.5
```
The Elexon coefficient gives the fraction of annual energy consumed in a given 30-minute period.

**`hhIndex` to time:**
`hhIndex × 30 minutes` from midnight UTC. Index 0 = 00:00–00:30, index 47 = 23:30–00:00.
